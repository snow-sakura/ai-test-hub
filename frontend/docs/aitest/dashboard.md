# 数据看板

数据看板统计卡片、图表、活动时间线 UI 规范。

## 页面布局

### 统计卡片区域

页面顶部展示四个统计卡片：

| 卡片 | 统计内容 | 图标 |
|------|----------|------|
| 项目总数 | 当前系统项目数量 | 项目图标 |
| 用例总数 | 测试用例总数 | 用例图标 |
| 执行次数 | 测试执行总次数 | 执行图标 |
| 通过率 | 测试平均通过率 | 通过率图标 |

### 图表区域

#### 用例趋势图

- 展示近期用例数量变化趋势
- 使用折线图或柱状图
- 支持按时间范围筛选

#### 执行趋势图

- 展示测试执行次数变化趋势
- 使用折线图
- 显示通过率趋势

#### 项目用例分布

- 展示各项目用例数量分布
- 使用饼图或横向柱状图
- 支持点击查看详情

### 活动时间线

- 展示近期系统活动
- 按时间倒序排列
- 显示活动类型、描述、时间
- 支持点击跳转

## API 接口

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/dashboard/overview | 获取概览统计 |
| GET | /api/v1/dashboard/trends | 获取趋势数据 |
| GET | /api/v1/dashboard/activity | 获取活动列表 |
| GET | /api/v1/dashboard/project-stats | 获取项目统计 |

## 状态管理

使用 Pinia store `dashboard.ts` 管理看板状态：

| State | Type | Description |
|-------|------|-------------|
| `overview` | object | 概览统计数据 |
| `trends` | object | 趋势数据 |
| `activity` | Activity[] | 活动列表 |
| `projectStats` | object[] | 项目统计 |
| `loading` | boolean | 加载状态 |

## 图表组件

- 使用 ECharts 或 Chart.js 作为图表库
- 图表响应式适配
- 添加图例和数据标签
- 支持 hover 交互

## 数据刷新

- 页面加载时自动刷新
- 支持手动刷新按钮
- 定时自动刷新（可选）
