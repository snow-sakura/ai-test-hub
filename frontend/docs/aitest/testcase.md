# 测试用例

测试用例列表、详情、编辑 UI 规范。

## 用例列表页面

### 页面布局

- 页面顶部：标题、搜索框、筛选条件、导入/导出按钮、新建按钮
- 主体区域：用例表格
- 支持按项目、优先级、状态筛选

### 用例表格

| 列 | 说明 |
|----|------|
| 编号 | 用例唯一标识 |
| 标题 | 用例标题，可点击跳转 |
| 模块 | 所属模块 |
| 优先级 | 高/中/低优先级标签 |
| 状态 | 待评审/已通过/已拒绝 |
| 类型 | 功能测试/性能测试/安全测试 |
| 创建时间 | 创建时间 |
| 操作 | 查看、编辑、删除、评审 |

## 用例详情页面

### 页面布局

- 页面顶部：用例标题、操作按钮（编辑、删除、评审）
- 基本信息：编号、模块、优先级、状态、类型
- 用例内容：前置条件、测试步骤、预期结果
- 附件区域：上传的附件列表
- 评论区域：用例评论列表

### 用例表单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本输入 | 是 | 用例标题 |
| 模块 | 下拉选择 | 是 | 所属模块 |
| 优先级 | 单选 | 是 | 高/中/低 |
| 类型 | 单选 | 否 | 功能测试/性能测试/安全测试 |
| 前置条件 | 文本域 | 否 | 测试前需要满足的条件 |
| 测试步骤 | 文本域 | 是 | 详细测试步骤 |
| 预期结果 | 文本域 | 是 | 预期的测试结果 |
| 附件 | 文件上传 | 否 | 相关附件 |

## API 接口

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/test-cases | 获取用例列表 |
| POST | /api/v1/test-cases | 创建用例 |
| GET | /api/v1/test-cases/{id} | 获取用例详情 |
| PUT | /api/v1/test-cases/{id} | 更新用例 |
| DELETE | /api/v1/test-cases/{id} | 删除用例 |
| GET | /api/v1/test-cases/{id}/comments | 获取评论列表 |
| POST | /api/v1/test-cases/{id}/comments | 添加评论 |
| GET | /api/v1/test-cases/{id}/attachments | 获取附件列表 |
| POST | /api/v1/test-cases/{id}/attachments | 上传附件 |

## 状态管理

使用 Pinia store `testcase.ts` 管理用例状态：

| State | Type | Description |
|-------|------|-------------|
| `testCases` | TestCase[] | 用例列表 |
| `currentTestCase` | TestCase | 当前用例 |
| `comments` | Comment[] | 评论列表 |
| `attachments` | Attachment[] | 附件列表 |
| `loading` | boolean | 加载状态 |

## 导入导出

- 支持 Excel 格式导入导出
- 导入时进行数据验证
- 导出包含用例完整信息
