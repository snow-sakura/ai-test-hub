# 用例评审

用例评审列表、详情、新建弹窗 UI 规范。

## 评审列表页面

### 页面布局

- 页面顶部：标题、搜索框、筛选条件、新建评审按钮
- 主体区域：评审列表表格
- 支持按项目、状态筛选

### 评审表格

| 列 | 说明 |
|----|------|
| 评审编号 | 评审唯一标识 |
| 项目名称 | 所属项目 |
| 用例数量 | 本次评审用例数 |
| 评审人 | 执行评审的人员 |
| 状态 | 待评审/评审中/已完成 |
| 创建时间 | 创建时间 |
| 操作 | 查看详情、继续评审 |

## 评审详情页面

### 页面布局

- 页面顶部：评审标题、操作按钮（继续评审、完成评审）
- 评审信息：项目名称、评审人、创建时间、状态
- 用例列表：待评审用例列表，支持分页
- 评审结果：已评审用例的结果统计

### 评审弹窗

- 宽度：600px
- 内容：用例详情、评审意见输入框、评审结果选择
- 评审结果：通过/拒绝/需要修改

## API 接口

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/reviews | 获取评审列表 |
| POST | /api/v1/reviews | 创建评审 |
| GET | /api/v1/reviews/{id} | 获取评审详情 |
| PUT | /api/v1/reviews/{id} | 更新评审 |
| DELETE | /api/v1/reviews/{id} | 删除评审 |
| POST | /api/v1/reviews/{id}/cases | 添加用例到评审 |
| PUT | /api/v1/reviews/{id}/cases/{case_id} | 评审单个用例 |
| GET | /api/v1/reviews/{id}/statistics | 获取评审统计 |

## 状态管理

使用 Pinia store `review.ts` 管理评审状态：

| State | Type | Description |
|-------|------|-------------|
| `reviews` | Review[] | 评审列表 |
| `currentReview` | Review | 当前评审 |
| `reviewCases` | ReviewCase[] | 评审用例列表 |
| `statistics` | object | 评审统计 |
| `loading` | boolean | 加载状态 |

## 自动评审

- AI 自动评审生成评审意见和分数
- 支持人工修改自动评审结果
- 自动评审结果可作为参考

## 评审统计

| 统计项 | 说明 |
|--------|------|
| 总用例数 | 评审用例总数 |
| 通过数 | 评审通过的用例数 |
| 拒绝数 | 评审拒绝的用例数 |
| 修改数 | 需要修改的用例数 |
| 通过率 | 通过用例占比 |
