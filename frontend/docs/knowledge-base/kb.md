# 知识库管理

知识库管理页面 UI 规范和 API 说明。

## 页面布局

### 页面头部

- 标题区域：标题、描述
- 搜索框：支持按名称搜索
- 视图切换：卡片视图/列表视图
- 新建按钮：创建新知识库

### 统计卡片

展示四个统计卡片：

| 卡片 | 统计内容 |
|------|----------|
| 知识库总数 | 当前系统知识库数量 |
| 文档总数 | 所有知识库文档总数 |
| 已启用 | 已启用的知识库数量 |
| 处理中 | 正在处理的知识库数量 |

### 知识库列表

#### 卡片视图

| 内容 | 说明 |
|------|------|
| 图标 | 知识库类型图标 |

| 文档数 | 包含的文档数量 |
| 创建时间 | 创建时间 |
| 操作按钮 | 管理文档、编辑、删除 |

#### 列表视图

| 列 | 说明 |
|----|------|
| 名称 | 知识库名称，可点击跳转 |
| 描述 | 简短描述 |
| 文档数 | 文档数量 |
| 状态 | 状态标签 |
| 创建时间 | 创建时间 |
| 操作 | 管理文档、编辑、删除 |

### 文档管理弹窗

- 文档列表表格
- 上传按钮
- 删除操作

## API 接口

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/knowledge-base | 获取知识库列表 |
| POST | /api/v1/knowledge-base | 创建知识库 |
| GET | /api/v1/knowledge-base/{id} | 获取知识库详情 |
| DELETE | /api/v1/knowledge-base/{id} | 删除知识库 |
| POST | /api/v1/knowledge-base/{id}/documents | 上传文档 |
| GET | /api/v1/knowledge-base/{id}/documents | 获取文档列表 |
| DELETE | /api/v1/knowledge-base/{id}/documents/{doc_id} | 删除文档 |

## 创建知识库表单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | 文本输入 | 是 | 知识库名称 |
| 描述 | 文本域 | 否 | 知识库描述 |
| 文档 | 文件上传 | 否 | 初始文档（可选） |

## 文档上传

### 支持格式

- .md (Markdown)
- .txt (纯文本)
- .pdf (PDF文档)
- .doc/.docx (Word文档)
- .xls/.xlsx (Excel表格)

### 处理流程

1. 上传文件
2. 文件解析
3. 文本提取
4. 切片处理
5. 向量化
6. 存入向量数据库

### 文档状态

| 状态 | 说明 |
|------|------|
| pending | 待处理 |
| processing | 处理中 |
| completed | 已完成 |
| failed | 处理失败 |

## 状态管理

使用 Pinia store 管理知识库状态：

| State | Type | Description |
|-------|------|-------------|
| `knowledgeBases` | KnowledgeBase[] | 知识库列表 |
| `currentKnowledgeBase` | KnowledgeBase | 当前知识库 |
| `documents` | Document[] | 文档列表 |
| `loading` | boolean | 加载状态 |

## RAG 集成

知识库可关联到 AI 聊天室：

1. 在聊天工具面板选择知识库
2. 发送消息时自动从知识库检索相关文档
3. AI 基于检索结果生成回复
4. 支持多知识库切换
